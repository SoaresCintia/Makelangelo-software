Rapport sur l'ajout de tests dans le projet Makelangelo-software

Par Larry Fotso Guiffo et Cíntia Dalila Soares

Tests Ajoutés

Les tests ajoutés se concentrent la classe TransformedImage :

Les tests pour la méthode canSampleAt(double x, double y) vérifient la capacité de cette dernière à déterminer si un point donné se trouve à l'intérieur ou à l'extérieur de l'image.

  • testImageInside() : vérifie que lorsqu’un point situé à l’intérieur des limites de l’image est fourni, la méthode renvoie true, indiquant que l'échantillonnage est possible.
  • testImageOutside() : vérifie que lorsqu’un point situé à l’extérieur des limites de l’image est fourni, la méthode renvoie false, indiquant que l'échantillonnage n'est pas possible.

Les tests pour la méthode sample(double x, double y) s’assurent que la méthode fonctionne correctement en différents scénarios de coordonnées d’échantillonnage.

  • testSampleZero() : vérifie que si des coordonnées inversées (gauche/droite et haut/bas) sont fournies et qu'un point situé en bas à gauche se trouve en dehors des limites de l'image, la méthode renvoie la valeur 0, indiquant que l'échantillonnage est invalide.

Les tests ajoutés se concentrent sur la classe NameThatColor :

Tests pour la méthode find(Long r, Long b , Long g) :

  • findColorTest() : Test si la methode retourne la bonne coleeur quand un code rbg lui est donne en parametre (eg: find(255,255,255) --> white).
  • findColorTest2() : Test si la methode retourne la bonne couleur quand un objet Color lui est passe en parametre(find(new Color(0, 0, 0)) --> black) .
  • findColorTest3() :Test si la methode retourne la bonne couleur quand un objet ColorRGB lui est passe en parametre(new ColorRGB(255, 0, 0)--> red)
  • findColorTest4() :Test si la methode retourne la bonne couleur quand un objet Long lui est passe en parametre(new Long(0xFF0000) --> red)

Tests pour la méthode le constructeur :

  • arrayColorContentTest() : Test si l'initialisation de la l'array list des coleur est bien reussit
